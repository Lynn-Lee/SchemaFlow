# 部署与安全配置

> 适用范围：SchemaFlow OSS Core 的静态 Docker/Nginx 自托管部署。

## Nginx 基础安全头

`default.conf.template` 已为静态页面和 `/config.js` 增加以下响应头：

- `Content-Security-Policy`：默认只允许同源脚本、静态资源和 worker；`connect-src 'self' https:` 仅允许同源请求和 HTTPS 自托管 AI Gateway endpoint，避免 XSS 后向任意明文 HTTP 端点外传数据。
- `X-Content-Type-Options: nosniff`：阻止浏览器 MIME sniffing。
- `Referrer-Policy: strict-origin-when-cross-origin`：跨站请求只发送 origin。
- `X-Frame-Options: SAMEORIGIN` 和 `frame-ancestors 'self'`：限制第三方页面嵌入。
- `Permissions-Policy: camera=(), microphone=(), geolocation=()`：默认禁用不需要的浏览器能力。

`/config.js` 额外设置 `Cache-Control: no-store`，避免运行时 endpoint、model 和开关配置在部署切换后被浏览器缓存。

## Self-hosted Gateway 配置边界

SchemaFlow 浏览器端不得接收长期模型 provider key。自托管 AI 能力只能通过以下方式接入：

- `OPENAI_API_ENDPOINT`：指向用户自托管 gateway 或兼容 OpenAI API 的代理 endpoint，不得包含 secret。
- `LLM_MODEL_NAME`：非敏感模型名称。
- `HIDE_SCHEMAFLOW_CLOUD`、`DISABLE_ANALYTICS`：非敏感运行时开关。

如果 gateway 使用了明文 HTTP、本地开发端口、WebSocket 或受限专用域名，需要在自托管镜像中审查并调整 CSP 的 `connect-src`。默认发布模板不再允许通配 `http:`，生产 gateway 应优先使用 HTTPS。

## 验证命令

Docker 镜像构建会在 builder 阶段使用 `NODE_OPTIONS=--max-old-space-size=4096`，避免 Vite 生产构建在容器默认 Node heap 限制下 OOM。

```bash
npm run test:ci -- src/lib/security/__tests__/nginx-security-headers.test.ts src/lib/security/__tests__/browser-key-exposure.test.ts
npm run build
docker build -t schemaflow-security-smoke .
```

如果本机 Docker daemon 不可用，至少保留 `npm run build`、安全测试和 Nginx 模板检查结果，并在验收记录中说明 Docker smoke 未执行原因。
