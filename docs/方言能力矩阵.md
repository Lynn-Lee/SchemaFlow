# ChartDB 方言能力矩阵

本文档记录 Phase 4 的 importer / exporter 插件化基线。矩阵只描述当前仓库已有能力和后续迁移目标，不引入账号登录、云端 diagram 存储、团队 Workspace、实时协作、在线评论、权限模型、计费、企业 SSO 或生产数据库直连。

## 1. 能力等级

| 等级 | 含义 | Phase 4 处理规则 |
| --- | --- | --- |
| `full` | 已有明确实现和测试，可作为迁移后默认能力 | 保持兼容，新增 fixture 和 warning regression |
| `partial` | 已有实现但存在语义缺口、兼容 fallback 或缺少 warning 输出 | 迁移时保留现有结果，同时补充 warnings、unsupportedObjects 和 sourceMap |
| `experimental` | 通过相邻方言或通用流程支持，行为未形成独立 contract | 先用 wrapper 标注风险，不作为无 warning 的稳定能力 |
| `unsupported` | 当前没有实现或不应自动转换 | 必须输出 unsupported syntax warning，不能静默丢弃 |

## 2. Import 能力矩阵

| 方言 | 当前入口 | import 状态 | 主要支持 | unsupported syntax | warning 规则 |
| --- | --- | --- | --- | --- | --- |
| PostgreSQL | `src/lib/data/sql-import/dialect-importers/postgresql`、`postgresql-dump` | `partial` | CREATE TABLE、schema、enum、自增类型、索引、外键、注释、pg_dump 检测 | RLS、policy、extension runtime 行为、trigger/function、view SQL body、复杂 domain/operator class | 迁移后对 extension、policy、function、trigger、无法映射的 index method 和被降级类型输出 warning |
| MySQL | `src/lib/data/sql-import/dialect-importers/mysql` | `partial` | CREATE TABLE、AUTO_INCREMENT、ENGINE、默认值、索引、view 基线 | trigger、procedure、partition、generated column 细节、charset/collation 精细语义 | 迁移后对 ENGINE、charset/collation 降级、generated column、procedure/trigger 输出 warning |
| MariaDB | MySQL importer fallback | `experimental` | 大多数 MySQL-compatible DDL | sequence、virtual column 差异、MariaDB-only engine/constraint 语义 | 迁移后必须标注使用 MySQL fallback，并对 MariaDB-only syntax 输出 warning |
| SQLite | `src/lib/data/sql-import/dialect-importers/sqlite` | `partial` | CREATE TABLE、INTEGER PRIMARY KEY、pragma-like dump 检测、基础外键 | trigger body、WITHOUT ROWID、CHECK 复杂表达式、partial index、virtual table | 迁移后对 virtual table、trigger、partial index、表达式 check 输出 warning |
| SQL Server | `src/lib/data/sql-import/dialect-importers/sqlserver` | `partial` | bracket identifier、schema、IDENTITY、NVARCHAR、默认值、外键、view 基线 | stored procedure、computed column 细节、filegroup、clustered/nonclustered 完整语义 | 迁移后对 stored procedure、filegroup、computed column、index option 降级输出 warning |
| Oracle | `src/lib/data/sql-import/dialect-importers/oracle` | `partial` | CREATE TABLE、NUMBER/VARCHAR2、schema-like owner、关系、常见类型 | package、sequence/trigger 自增组合、tablespace、synonym、materialized view | 迁移后对 sequence/trigger、tablespace、package、synonym、materialized view 输出 warning |
| CockroachDB | PostgreSQL importer fallback | `experimental` | PostgreSQL-compatible table、index、relationship | interleave/locality、computed/sharded index、zone config、changefeed | 迁移后必须标注使用 PostgreSQL fallback，并对 CockroachDB-only syntax 输出 warning |
| ClickHouse | 无独立 importer | `unsupported` | 暂无稳定 import | engine、partition/order by、materialized view、distributed table | 迁移前遇到 ClickHouse DDL 必须输出 unsupported syntax warning，不应伪装为 PostgreSQL/MySQL |
| DBML | 独立 DBML flow | `partial` | table、field、relationship、enum 语义 | 非 DBML SQL statement、复杂数据库特有选项 | 迁移后纳入统一 pipeline，DBML 无法表达的数据库特性输出 warning |

## 3. Export 能力矩阵

| 方言 | 当前入口 | export 状态 | 主要支持 | unsupported syntax | warning 规则 |
| --- | --- | --- | --- | --- | --- |
| PostgreSQL | `src/lib/data/sql-export/export-per-type/postgresql.ts` | `partial` | table、field、relationship、schema、comment、enum/composite custom type | policy、extension、trigger、function、advanced index option | 迁移后输出 unsupportedObjects，并在风险等级中标注 schema/custom type 是否完整 |
| MySQL | `src/lib/data/sql-export/export-per-type/mysql.ts`、PostgreSQL to MySQL mapping | `partial` | table、field、relationship、基础 index、PostgreSQL deterministic cross-dialect | check constraint 差异、enum/composite 降级、schema 到 database 语义差异 | 迁移后对类型映射、constraint 降级和 schema 转换输出 warning |
| MariaDB | MySQL exporter fallback | `experimental` | MySQL-compatible table 和 relationship | MariaDB-only sequence、engine 差异、虚拟列细节 | 迁移后必须标注 fallback，并对 MariaDB-only unsupported syntax 输出 warning |
| SQLite | `src/lib/data/sql-export/export-per-type/sqlite.ts` | `partial` | table、field、relationship、基础 index | schema、部分 alter、enum/custom type、复杂 check | 迁移后对 schema/custom type/alter 降级输出 warning |
| SQL Server | `src/lib/data/sql-export/export-per-type/mssql.ts`、PostgreSQL to MSSQL mapping | `partial` | table、field、relationship、schema、基础 index | filtered index、computed column、filegroup、stored procedure | 迁移后对类型映射、identity/default 差异和 index option 降级输出 warning |
| Oracle | 无独立 exporter | `unsupported` | 暂无稳定 deterministic export | sequence/identity 策略、schema owner、tablespace、package | P4-003 前必须返回 unsupported export warning，不能静默用 PostgreSQL 输出替代 |
| CockroachDB | PostgreSQL exporter fallback | `experimental` | PostgreSQL-compatible table、relationship | locality、interleave、zone config、regional table | 迁移后必须标注 fallback，并输出 CockroachDB-only warning |
| ClickHouse | 无独立 exporter | `unsupported` | 暂无稳定 deterministic export | engine、partition/order by、primary key 语义、materialized view | P4-003 前必须返回 unsupported export warning |
| DBML | DBML export flow | `partial` | table、relationship、enum | vendor-specific table option、trigger、function、policy | 迁移后纳入统一 exporter result，无法表达项进入 warnings |

## 4. 统一 contract 要求

Phase 4 后续代码任务必须让每个 dialect adapter 输出同构结果：

- `ImportResult`：包含 `diagram`、`warnings`、`unsupportedObjects`、`sourceMap` 和原始 `sourceDialect`。
- `ExportResult`：包含 `output`、`warnings`、`unsupportedObjects`、`riskLevel` 和目标 `targetDialect`。
- `warning` 必须包含 `code`、`severity`、`message`、可选 `statementType`、可选 `sourceRange`。
- `unsupportedObjects` 必须记录 object type、object name、原因和是否被忽略。
- `sourceMap` 至少能把 table、field、relationship、index 和 custom type 映射回源 statement 或源行号。
- `riskLevel` 使用 `low`、`medium`、`high`。出现静默丢弃风险时不得低于 `medium`；出现会生成错误 SQL 的已知风险时不得低于 `high`。

## 5. 迁移顺序

1. `CHARTDB-P4-001`：创建 `src/dialects/common` contract、result type 和 capability matrix type，旧 importer/exporter 通过 adapter 兼容。
2. `CHARTDB-P4-002`：先迁移 PostgreSQL importer，覆盖 pg_dump、enum、extension、RLS/policy warning 和 regression fixtures。
3. `CHARTDB-P4-003`：迁移 MySQL、MariaDB、SQLite、SQL Server、Oracle wrapper，明确 fallback 与 unsupported warning。
4. `CHARTDB-P4-004`：统一 deterministic exporter result，覆盖 PostgreSQL、MySQL、SQLite、SQL Server、DBML 和 unsupported Oracle/ClickHouse。
5. `CHARTDB-P4-005`：把 capability matrix 自动生成到文档，确保 DBML、ClickHouse、CockroachDB 等 experimental/unsupported 状态不会被 UI 误报成 full support。
