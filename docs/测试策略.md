# 测试策略

> 对应任务：`SCHEMAFLOW-P7-002`
> 适用范围：本地开发、CI、发布和自动化 dispatcher。

## 1. 默认门禁

每个源码、配置或用户可见文档切片完成后默认运行：

```bash
npm run lint
npm run test:ci
npm run build
git diff --check
```

涉及安全依赖、AI key、Docker、Nginx、Markdown 渲染或发布 workflow 时额外运行：

```bash
npm audit --omit=dev --audit-level=high
rg -n "VITE_OPENAI_API_KEY|OPENAI_API_KEY|window\\.env|rehype-raw|dangerouslySetInnerHTML" src Dockerfile default.conf.template .github
```

## 2. TDD 要求

代码行为变更必须先补失败测试或可重复失败检查，再实现最小修复，最后确认绿灯。文档切片可以使用内容检查作为红灯，例如缺少文件、README 链接或关键验收 token。

## 3. 核心测试范围

- `src/schema-core/**/__tests__`：领域模型、command、validation 和 history metadata。
- `src/storage/**/__tests__`：Dexie schema、repository、transaction、backup/restore。
- `src/dialects/**/__tests__`：方言 capability、import/export wrapper、warning 和 unsupported object。
- `src/features/import/**`：import preview summary、parser fallback 和确认前不写入数据。
- `src/lib/security/**/__tests__`：浏览器密钥暴露、Markdown 安全、Nginx security headers。
- `src/workers/**/__tests__`：worker client、fallback 和错误结构。

## 4. UI Smoke

涉及 UI 的任务需要在浏览器中至少覆盖桌面和移动尺寸：

- 关键入口可见。
- 无明显横向溢出。
- 核心按钮、dialog、toast 和空状态可操作。
- console 无严重 error。

## 5. 发布 Smoke

发布 workflow 必须在正式 multi-arch push 前执行 Docker smoke：

- 本地构建 `linux/amd64` 镜像。
- 启动容器并检查 `/` 和 `/config.js`。
- 验证 CSP、`X-Content-Type-Options`、`Referrer-Policy`、`X-Frame-Options`、`Permissions-Policy`。
- 验证 `/config.js` 包含 `Cache-Control: no-store`。

Docker daemon、基础镜像或网络不可用时不得自动放行发布。

## 6. 依赖与安全

Production high/critical advisory 必须阻断发布。Low/moderate advisory 可以保留，但必须在 `docs/安全风险登记.md` 中说明影响范围、当前缓解和后续处理策略。

## 7. 性能记录

涉及性能任务时，需要记录 `npm run build` 输出中的主要 chunk 变化，尤其是入口 chunk、Monaco、editor-page、模板数据和 worker 相关 chunk。
