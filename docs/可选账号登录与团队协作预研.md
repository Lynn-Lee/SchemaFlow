# 可选账号登录与团队协作预研

> 任务：`CHARTDB-P8-000`
> 定位：Phase 8 只做 Cloud/Team 产品边界和接口预研，不新增登录代码。

## 1. 结论

ChartDB 的默认产品形态仍是 OSS Core 本地优先模式：不登录仍可完整使用 OSS Core，用户无需账号、无需数据库密码，也不需要连接任何 ChartDB 服务端。

可选 Cloud/Team 应作为独立增强产品线推进。它可以提供账号登录、Workspace、Team、云同步、分享权限、版本历史和审计能力，但不能成为本地模式的前置依赖，也不能改变现有导入、编辑、导出、备份和恢复流程。

本轮只定义边界，不实现登录、OAuth、SSO、Workspace API、云端 diagram 存储、实时协作、评论、计费或权限代码。

## 2. OSS Core 保持不变

硬约束：

- 不登录仍可完整使用 OSS Core。
- 本地 diagram 不自动上传。
- 用户主动开启云同步后，才允许把指定 diagram 写入云端。
- 本地历史 diagram 不因登录而自动迁移到云端。
- AI Disabled、BYOK Session 和 Self-hosted Gateway 的安全边界不因 Cloud/Team 改变。
- 本地备份文件、DBML、SQL、PNG、SVG 和 Markdown 导出能力不依赖账号。

本地模式数据边界：

- 默认数据源仍是浏览器 IndexedDB。
- Smart Query 仍由用户在自己的数据库环境执行。
- ChartDB 前端不直接连接生产数据库。
- 导入内容、diagram 内容、note 和导出结果默认只在浏览器内处理。

## 3. 可选 Cloud/Team 能力范围

Cloud/Team 可以预留以下能力，但必须独立于 OSS Core 发布：

- 账号登录：Email/password、OAuth、SSO 只作为后续方案候选。
- Workspace：组织团队空间、成员列表和默认策略。
- Team：按团队组织 diagram、模板和分享链接。
- Diagram 云同步：用户显式选择同步目标，支持本地副本和云端副本分离标识。
- 分享权限：只读、可编辑、可复制三类最小权限。
- 版本历史：记录 diagram 版本、修改来源和恢复点。
- 审计日志：记录登录、成员变更、权限变更、分享链接创建、云同步、导出和删除。
- 数据治理：云端数据导出、数据删除、租户删除和保留期策略。

暂不纳入：

- 实时多人协作。
- 在线评论和审批流。
- 计费。
- 企业策略中心。
- 生产数据库直连。

## 4. 数据与接口边界

Cloud/Team 不应复用本地 IndexedDB 作为权威状态。建议后续单独定义服务端 Domain Model：

- `Account`：登录主体，不进入 OSS Core bundle 的默认运行路径。
- `Workspace`：租户级边界。
- `Team`：Workspace 内的协作组。
- `CloudDiagram`：云端 diagram 元数据和当前版本指针。
- `CloudDiagramVersion`：不可变版本快照。
- `ShareLink`：分享链接、权限和过期时间。
- `AuditLog`：服务端 AuditLog，记录 actor、target、action、timestamp、requestId 和 metadata。

接口 contract 应满足：

- 所有写入 API 必须带租户上下文。
- 所有 diagram 读取和写入必须校验 Workspace / Team 权限。
- 云同步 API 必须区分 create、pull、push、fork 和 disconnect。
- 删除 API 必须区分软删除、恢复、永久删除和数据导出前置检查。
- 导出 API 必须支持用户下载自己的云端数据。

## 5. 权限和审计策略

最小权限模型：

- Owner：管理 Workspace、成员、Team、分享链接、删除和导出。
- Maintainer：管理 Team diagram、版本和分享链接。
- Editor：编辑指定 diagram。
- Viewer：只读查看指定 diagram。

权限 negative tests 必须覆盖：

- 非成员不能读取 Workspace diagram。
- Viewer 不能编辑、删除、创建分享链接或导出私有 Workspace 数据。
- Editor 不能修改 Team 成员或 Workspace 设置。
- 被移除成员不能继续通过旧会话访问云端 diagram。
- 过期分享链接不能访问 diagram。
- 跨租户 ID 枚举不能读写其它租户数据。
- 断开云同步后，本地 diagram 不再向云端写入。

AuditLog 必须覆盖：

- 登录和登出。
- 成员邀请、接受、移除和角色变更。
- diagram 创建、同步、读取、导出、删除和恢复。
- 分享链接创建、权限变更、过期和撤销。
- 权限拒绝事件和跨租户访问尝试。

## 6. 隐私与合规边界

Cloud/Team 一旦启用，会把用户 schema、note、diagram layout 和版本历史保存到服务端，因此必须在产品和 API 上提供显式确认。

必须提供：

- 启用云同步前的确认说明。
- diagram 级同步开关。
- 云端数据导出。
- 云端数据删除。
- Workspace 删除和保留期说明。
- 敏感字段脱敏建议。
- 企业 SSO 或审计导出只作为后续可选项。

不得提供：

- 静默上传本地 diagram。
- 登录后自动同步全部历史 diagram。
- 把 API key、数据库密码或 Smart Query 连接凭据写入云端。
- 在 OSS Core 默认构建中硬依赖 Cloud/Team SDK。

## 7. 后续技术预研问题

`CHARTDB-P8-001` 需要继续回答：

- 后端框架选型：NestJS、Fastify 或其它 TypeScript 服务端框架。
- 数据库选型：PostgreSQL schema、租户隔离方式和 migration 策略。
- Auth 选型：自建 Auth、OAuth provider、SSO/SAML/OIDC 的分层边界。
- 同步策略：last-write-wins、版本分支、冲突检测或人工 merge。
- API contract：REST、RPC 或 GraphQL 的取舍。
- 部署边界：OSS Core、Self-hosted Cloud 和商业 Cloud 如何隔离。

## 8. 验收

- 明确不登录仍可完整使用 OSS Core。
- 明确本地 diagram 不自动上传，云同步必须用户主动开启。
- 明确 Workspace、Team、权限、AuditLog、租户隔离、数据导出、数据删除和 negative tests。
- 明确本轮不新增登录代码，不改变 OSS Core 默认本地优先模式。
