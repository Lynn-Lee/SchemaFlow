# 安全风险登记

> 最近更新：2026-07-01
> 适用范围：ChartDB OSS Core 本地优先模式与静态 Docker/Nginx 自托管部署。

## Phase 1 审查结论

`CHARTDB-P1-005` 已完成密钥、AI、Markdown、Docker/Nginx 和生产依赖 advisory 审查。当前没有阻断 Phase 2 的 high 或 critical 风险。

## High 风险

当前状态：无开放项。

已关闭项：

- 浏览器端长期 OpenAI API key 暴露：`Dockerfile` 不再写入 `VITE_OPENAI_API_KEY`，`default.conf.template` 的 `/config.js` 不再输出 `OPENAI_API_KEY`。
- Note Markdown raw HTML/XSS 风险：`rehype-raw` 已移除，Note Markdown 不再把 raw HTML 渲染为真实 DOM，危险链接协议不可点击。
- Docker/Nginx 缺少基础安全头：静态页面和 `/config.js` 均返回 CSP、nosniff、Referrer-Policy、X-Frame-Options 和 Permissions-Policy。

## Medium 风险

### 生产依赖 `dompurify` advisory，经 `monaco-editor` 引入

- 严重级别：moderate。
- 影响路径：`monaco-editor -> dompurify`。
- 当前缓解：ChartDB Note Markdown 不使用 DOMPurify 清洗 raw HTML；P1-003 已禁用 raw HTML 渲染并增加恶意 Markdown 回归测试。生产 high gate 通过。
- 暂不修复原因：npm 给出的修复路径需要对 `monaco-editor` 做破坏性版本变更，风险超过本轮安全审查切片。
- 后续处理：进入 Phase 6 性能/Monaco lazy 拆分或独立依赖治理任务时，评估 Monaco 升级、降级或替换方案，并保留编辑器 smoke。

## Low 风险

### AI SDK provider-utils 资源消耗 advisory

- 严重级别：low。
- 影响路径：`ai`、`@ai-sdk/openai`、`@ai-sdk/gateway`、`@ai-sdk/provider-utils`。
- 当前缓解：P1-001 暂停非 deterministic AI-assisted SQL export fallback；P1-002 增加 AI mode gate，默认 Disabled，BYOK/Gateway 请求前必须显式确认 schema transfer。
- 暂不修复原因：npm 给出的修复路径需要 `ai` / `@ai-sdk/openai` semver-major 升级，且真实模型调用尚未恢复。
- 后续处理：在设置中心和 Self-hosted Gateway 真正落地前重新评估 AI SDK 版本，保留默认 Disabled 和不持久化 key 的安全 contract。

### `/config.js` 暴露 `window.env`

- 严重级别：low。
- 影响路径：`default.conf.template`。
- 当前解释：`window.env` 仅输出 `OPENAI_API_ENDPOINT`、`LLM_MODEL_NAME`、`HIDE_CHARTDB_CLOUD` 和 `DISABLE_ANALYTICS` 等非敏感运行时配置，不输出 provider key 或 secret。
- 当前缓解：`/config.js` 设置 `Cache-Control: no-store`，并返回与静态页面一致的基础安全头。
- 后续处理：保留为静态自托管 runtime config 机制；新增配置项前必须确认不包含 secret。

## Phase 2 准入结论

Phase 1 退出门槛已满足：

- `npm audit --omit=dev --audit-level=high` 通过，生产 high/critical advisory 为 0。
- 安全扫描不再命中 `VITE_OPENAI_API_KEY`、`OPENAI_API_KEY`、`rehype-raw` 或 `dangerouslySetInnerHTML` 的源码/配置风险。
- 唯一保留命中 `window.env` 已解释为非敏感 runtime config。
- Markdown XSS 回归测试和 Nginx 安全头测试已建立。

允许下一轮进入 `CHARTDB-P2-000`，编写 Phase 2 Schema Core 与 Command 架构执行清单。
