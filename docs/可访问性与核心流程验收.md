# SchemaFlow 可访问性与核心流程验收

> 版本：v1.0
> 日期：2026-07-02
> 适用阶段：Phase 5 产品体验与可访问性
> 覆盖任务：`SCHEMAFLOW-P5-001` 到 `SCHEMAFLOW-P5-004`

## 1. 验收目标

Phase 5 的目标不是新增账号、云存储或协作能力，而是降低本地优先 SchemaFlow 的首次使用门槛，并让核心流程可以被键盘、屏幕阅读器和移动端用户稳定完成。

本矩阵作为后续 Phase 5 代码任务的验收依据。任一任务若修改首次进入、Smart Query、导入、设置、Dialog、Button、Radio、Monaco 或移动端布局，都必须回归本文件中相关流程。

## 2. 范围边界

允许范围：

- 首次进入流程的信息架构、入口拆分和错误提示。
- Smart Query 复制 SQL、粘贴结果、解析 preview 和确认导入。
- icon button、radio、dialog、toolbar、Monaco editor 的可访问名称和键盘路径。
- 画布选中节点后的最小键盘路径：Delete/Backspace 删除，方向键微调位置。
- 设置中心的信息归位，包括本地隐私、AI mode、导出默认项、数据导出和清理入口。
- 桌面和移动端布局 smoke。

禁止范围：

- 不新增账号登录、注册、OAuth、SSO。
- 不新增云端 diagram 存储、团队 workspace、实时协作或在线评论。
- 不在前端持久化 AI API key。
- 不绕过 Phase 2 command 或 Phase 3 repository 直接写多处状态源。
- 不把画布节点拖拽、连线创建和复杂图形编辑要求为完整键盘等价操作；这部分属于已知限制，需后续独立产品设计和交互任务承接。

## 3. 核心流程验收矩阵

| 流程 | 桌面验收 | 移动验收 | 键盘验收 | 可访问性验收 | 失败状态验收 |
| --- | --- | --- | --- | --- | --- |
| 首次进入 | 首屏展示导入现有数据库、新建空白图、查看模板三个入口；主动作在 1280px 宽度内无需滚动可见。 | 390px 宽度下主动作在第一屏可见；底部操作区不遮挡内容。 | Tab 顺序先到数据库选择，再到启动方式，再到继续或取消。 | 数据库选择 radio 有唯一 accessible name；当前选择可被读出。 | 未选择数据库或启动方式时，继续按钮提示原因，不创建半成品 diagram。 |
| Smart Query | 清晰展示选择数据库、复制 SQL、粘贴结果、校验摘要、导入 preview 五步。 | 步骤内容纵向排列，复制和继续按钮不产生横向溢出。 | 可用键盘复制 SQL、切换步骤、聚焦粘贴区、确认 preview。 | 复制按钮、返回、继续、关闭按钮有 `aria-label` 或可见文本；Monaco 区域有用途名称。 | 空输入、JSON 解析失败、方言不支持、unsupportedObjects 均显示可操作提示。 |
| 导入 Preview | tables、relationships、customTypes、warnings 和 unsupportedObjects 数量清晰可见。 | Preview 列表可滚动，确认按钮保持可见。 | Preview dialog 可关闭、返回编辑、确认导入。 | warning 与 unsupportedObjects 不只依赖颜色表达。 | 用户确认前不写入 IndexedDB；解析失败不清空原输入。 |
| 编辑画布 | Toolbar、建表、字段、关系和导出入口位置稳定。 | 画布 controls 不遮挡底部导航或 Dialog 操作区。 | 可通过键盘打开核心 Dialog；Dialog 关闭后焦点返回触发按钮；已选中节点可用 Delete/Backspace 删除、方向键微调位置。 | icon button 全部有名称；画布容器有可访问名称；destructive action 有明确确认文案。 | command validation error 能说明失败对象和原因。 |
| 设置 | theme、language、AI mode、导出默认项、本地数据管理集中在设置中心。 | 设置页分组可扫描，危险操作不与普通开关并列。 | 可用键盘切换 tab、开关和 select。 | 设置分组有 heading；开关有 label；危险操作有确认 dialog。 | localStorage 不可用时显示降级说明；设置不保存 secret。 |

## 4. 自动与人工检查

每个 Phase 5 代码任务默认运行：

```bash
npm run lint
npm run test:ci
npm run build
git diff --check
```

涉及 UI 的任务额外运行本地浏览器 smoke：

- 桌面：1280x720，打开主流程并检查 console 无严重错误。
- 移动：390x844，检查无明显横向溢出，主动作可见。

涉及可访问性命名的任务额外执行：

```bash
rg -n "<Button|<button|aria-label|role=\"dialog\"|RadioGroup|Monaco" src/components src/dialogs src/features src/pages
```

人工检查至少覆盖：

- Tab 顺序是否符合视觉顺序。
- Dialog 打开后初始焦点是否落在有效操作或标题附近。
- Dialog 关闭后焦点是否返回触发按钮。
- Disabled 控件是否解释原因。
- 错误提示是否说明下一步，而不是只显示技术异常。

## 6. Phase 5 退出标准

Phase 5 通过前必须满足：

- `SCHEMAFLOW-P5-001` 到 `SCHEMAFLOW-P5-004` 均完成并通过默认门禁。
- 桌面和移动 smoke 覆盖首次进入、Smart Query、导入 preview、设置中心。
- 核心 icon button、radio、dialog 和 Monaco editor 已有可访问名称或清晰可见文本。
- 键盘可以完成导入、新建空白图、打开设置、导出结果四条核心路径。
- 没有新增账号、云端存储、团队权限、持久化 AI secret 或远端 schema 上传能力。
