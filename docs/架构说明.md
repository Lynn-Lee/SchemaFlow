# SchemaFlow 架构说明

> 对应任务：`SCHEMAFLOW-P7-002`
> 适用范围：SchemaFlow OSS Core 本地优先模式。

## 1. 产品边界

SchemaFlow 当前默认运行在浏览器内，不要求账号、数据库密码或生产数据库直连。用户在自己的数据库环境中执行 Smart Query，将脱敏后的结构化结果、SQL、DBML 或模板导入 SchemaFlow；diagram 默认保存在浏览器 IndexedDB。

Phase 0 到 Phase 7 不实现账号登录、云端 diagram 存储、团队 Workspace、实时协作、在线评论、权限模型、计费、企业 SSO 或生产数据库直连。

## 2. 前端入口

- `src/main.tsx`：React 应用入口。
- `src/router.tsx`：本地路由定义。
- `src/pages/editor-page`：主要 ERD 编辑画布、侧边栏、导入导出和上下文操作入口。
- `src/context/schemaflow-context`：当前 diagram 的 UI 状态、command 调用和历史兼容层。
- `src/context/storage-context`：向 UI 提供 storage repository 与 transaction 能力。

UI 层不应直接绕过 command、repository 或 transaction service 去批量改写多个状态源。

## 3. 领域模型与命令

- `src/schema-core/model`：Diagram、Table、Field、Relationship、Area、Note、CustomType 等领域类型出口。
- `src/schema-core/commands`：table、field、relationship、area、note、custom type 和 command history 的纯函数 contract。

`schema-core` 不依赖 React、Dexie、Monaco 或浏览器 UI。高风险 command 需要返回 validation error、warning 或 history metadata，不能静默丢弃用户数据。

## 4. Storage 层

- `src/storage/db`：Dexie instance、schema version 和 migration 声明。
- `src/storage/repositories`：diagram、table、relationship、dependency、area、custom type、note、filter 和 config 的 CRUD 边界。
- `src/storage/transactions`：diagram 级创建、删除、替换和恢复事务。
- `src/storage/backup`：`schemaflow.backup` 文件导出、导入和校验。

Storage 层只处理本地 IndexedDB 和 backup 文件，不承担 toast、路由跳转、权限或云同步职责。

## 5. Dialect 与导入导出

- `src/dialects/common`：统一 `ImportResult`、`ExportResult`、capability、warning、unsupported object 和 source map contract。
- `src/dialects/postgresql`、`mysql`、`mariadb`、`sqlite`、`sqlserver`、`oracle`、`dbml`：各方言 wrapper、capability 和 warning 规则。
- `src/features/import`：导入 preview 摘要和确认前校验流程。

方言 adapter 必须保留 unsupported 或降级语义；不能把不支持的对象静默映射成看似成功的结果。

## 6. Worker 与性能边界

- `src/workers/import-worker`：SQL、DBML 和 metadata preview parse 的 module worker。
- `src/workers/layout-worker`：Area auto arrange 和 move-to-area 布局计算。
- `src/workers/worker-client.ts`：worker 调用和 fallback contract。

Worker 不可用时允许回退主线程实现，但必须返回同构结果，避免 UI 流程分叉。

## 7. AI 与运行时配置

AI-assisted export 默认 Disabled。BYOK Session 只允许把用户密钥保存在当前运行时内存；Self-hosted Gateway 只允许浏览器接收非敏感 endpoint 和 model name。Docker、Nginx 和 `/config.js` 不得输出长期模型 provider secret。

相关细节见 `docs/安全模型与AI边界.md` 和 `docs/部署与安全配置.md`。

## 8. 发布边界

发布前必须通过 lint、test、build、production audit、`git diff --check`、安全扫描和 Docker smoke。Docker daemon、基础镜像或网络不可用时不得自动放行发布，必须记录 blocker 或人工补跑要求。
